# 基于Qwen3-VL的校园网违规图片智能审查系统与轻量级部署研究
# 面向高并发场景的“CNN-VLM”级联图像审核系统设计与轻量化方案分析

## 选题背景及意义

## 核心技术栈
- 基座模型：Qwen3-VL-7B-Instruct (或根据显存选择 2B / 7B 版本)。Qwen3-VL在OCR识别、长文本理解和复杂场景推理上表现优异。
- 推理框架：vLLM (高吞吐) 或 Ollama / LMStudio (易用性)，配合 bitsandbytes 进行量化。
- 后端服务：FastAPI + Celery (异步任务队列，处理上传峰值)。
- 前端展示：Vue3 + Element Plus (审核后台)。
- 部署环境：Docker + NVIDIA Triton Inference Server (可选，用于生产级优化)。

## 轻量级部署可行性分析（核心研究点）
这是本毕设的难点与创新点。Qwen3-VL-7B 原生需要较大显存，需通过以下技术手段实现“轻量级”：
### 4.1 量化技术 (Quantization)
方案：使用 `AWQ` (Activation-aware Weight Quantization) 或 `GGUF` (llama.cpp架构) 将模型量化为 INT4 甚至 INT8。

### 4.2 模型蒸馏与剪枝 (Distillation & Pruning) - 进阶选项
方案：如果7B仍过大，可研究使用 Qwen3-VL-7B 作为教师模型，蒸馏出一个更小的学生模型（如基于 MobileVL 或微调一个小参数量的VLM），专门针对校园违规场景训练。
注意：对于本科生毕设，直接使用量化后的 `Qwen3-VL-2B` 或 7B-INT4 通常已足够证明可行性，蒸馏难度较大，可作为“未来展望”。
### 4.3 推理加速
KV Cache 优化：利用 vLLM 的 PagedAttention 技术，提高并发处理能力。
批处理 (Batching)：将短时间内上传的多张图片合并为一个 Batch 进行推理，降低延迟。
### 4.4 硬件成本估算表 (示例)

### 潜在风险与应对
风险：模型产生幻觉（误判正常图片为违规）。
应对：设置“人机回环”机制，低置信度或特定类别强制人工复核；优化Prompt增加思维链（CoT）。
风险：推理速度过慢，无法应对课间高峰期。
应对：引入消息队列（RabbitMQ/Kafka）削峰填谷；采用更小参数版本（2B）作为初筛，7B作为精筛。
风险：显存不足。
应对：严格使用 INT4 量化；使用 CPU Offload 技术（虽然会慢，但能跑起来做演示）

### 给本科生的特别建议
不要从头训练模型：本科生毕设时间和算力有限，重点应放在应用落地、Prompt工程、量化部署和系统集成上，而不是预训练或全量微调。
数据隐私：在论文中强调系统设计时考虑了用户隐私（如图片处理后立即删除、不存储原图等），这是网络工程类毕设的加分项。
量化对比：一定要有一章专门对比“未量化”vs“量化后”的资源消耗，这是体现“轻量级部署可行性”的最有力证据。
